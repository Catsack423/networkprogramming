# Network Programming & Advanced Networking Labs

โครงการรวบรวมแบบฝึกหัดและการทดลองด้านการเขียนโปรแกรมเครือข่าย (Network Programming) ตั้งแต่ระดับพื้นฐานไปจนถึงเทคโนโลยีเครือข่ายขั้นสูงที่เลียนแบบธรรมชาติและควอนตัม


| สัปดาห์ | หัวข้อหลัก | รายละเอียดการทดลอง |
| :--- | :--- | :--- |
| **Week 1** | **TCP Server** | สร้าง Chat Server และระบบ Server ขั้นพื้นฐานด้วย TCP |
| **Week 2** | **UDP Unicast** | การรับส่งข้อมูลเซ็นเซอร์ผ่าน UDP ในรูปแบบ Unicast |
| **Week 3** | **LAN Discovery** | การค้นหาบริการและอุปกรณ์ในเครือข่ายท้องถิ่นผ่าน UDP Broadcast |
| **Week 4** | **UDP Multicast** | ระบบ Publish/Subscribe สำหรับการสื่อสารแบบกลุ่ม |
| **Week 5** | **P2P Networking** | การรับส่งข้อมูลแบบ Peer-to-Peer โดยไม่ต้องผ่านเซิร์ฟเวอร์กลาง |
| **Week 6** | **MANET** | เครือข่ายไร้สายเมช (Ad-hoc Mesh) สำหรับสถานการณ์ฉุกเฉิน |
| **Week 7** | **Store & Forward** | ระบบการส่งข้อมูลในเครือข่ายที่มีการเชื่อมต่อไม่สม่ำเสมอ (เช่น เครือข่ายระหว่างดาวเคราะห์) |
| **Week 8** | **Opportunistic Networking** | เครือข่ายที่อาศัยโอกาสและการเคลื่อนที่ของโหนดในการส่งต่อข้อมูล |
| **Week 9** | **Bio-inspired Networking** | การเลียนแบบการทำงานของระบบประสาท (BCI Simulation) และการเชื่อมต่อแบบชีวภาพ |
| **Week 10** | **Quantum Networking** | การจำลองการสื่อสารที่ปลอดภัยด้วยแนวคิดควอนตัมและการจัดการสถานะ (State) |

## 🛠️ ส่วนขยายและเนื้อหาเพิ่มเติม (Extension Branches)

> [!IMPORTANT]
> สำหรับExtension ทั้งหมด จะไม่ได้อยู่รวมในกิ่งหลัก (main) แต่จะถูกแยกออกเป็นกิ่งย่อย (Branch) ตามสัปดาห์นั้น ๆ



## 🚀 เริ่มต้นใช้งาน (Getting Started)

1. ตรวจสอบว่าระบบมี **Python 3.x** ติดตั้งอยู่
2. เลือกสัปดาห์ที่ต้องการศึกษาในรูปแบบโฟลเดอร์ เช่น `week01-tcp-server-basic`
3. เข้าไปในโฟลเดอร์นั้นและรันสคริปต์หลัก (เช่น `server.py` หรือ `node.py`)
4. เปลี่ยน config.py ให้portตรงกับportที่รันในเครื่องได้เลย
